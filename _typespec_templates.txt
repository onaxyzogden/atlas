OLOS Project Type Templates
Developer specification for Stage Zero configuration, domain priorities, map overlays, starter objectives, and stage handoffs

[TABLE]
Core Product Rule / Project Type Templates are Stage Zero presets. They do not replace Observe, Plan, or Act. They configure which domains, overlays, questions, objectives, warnings, dashboards, and handoff packages OLOS should emphasize for a particular land-based project.
[/TABLE]

1. Purpose
This document defines reusable project type templates for OLOS. When a user chooses a land-based project type in Stage Zero, the selected template should preload a guided workflow across Observe, Plan, and Act. The goal is to prevent the user from seeing every possible domain and overlay at once while still preserving the universal OLOS architecture.
A project type template should behave like a configuration layer: it weights domains, chooses default overlays, asks relevant setup questions, loads starter objectives, checks for red flags, and shapes dashboards. Users should still be able to customize, combine, or override templates.
2. How Templates Fit the OLOS Architecture

[TABLE]
Layer | Role in OLOS
Stage Zero | Defines the project type, intent, constraints, non-negotiables, expected outputs, and initial feasibility questions.
Project Type Template | Preloads the domain priority model, relevant overlays, starter objectives, dashboards, and risk checks.
Stages | Observe gathers evidence, Plan makes decisions, and Act executes, verifies, maintains, and feeds learning back into the system.
Domains | Universal areas of land intelligence that remain consistent across project types.
Objectives | Focused pieces of work inside a domain. Objectives determine the checklist and map view state.
Overlays | Map lenses loaded only when useful for the selected domain + objective.
Handoffs | Structured outputs passed from Stage Zero to Observe, Observe to Plan, and Plan to Act.
[/TABLE]


[TABLE]
Developer Implementation Note / The template should not hard-code a single path. It should create a recommended path with editable defaults. A user may start with Homestead, add a Market Garden component, then later activate Agritourism or Education objectives.
[/TABLE]

3. Template Object Schema

[TABLE]
Field | Purpose
template_id | Stable machine-readable ID such as PT-HOMESTEAD or PT-REGEN-FARM.
name | Human-readable template name.
summary | One-paragraph explanation of what this template is for.
best_fit | The user/project scenario this template serves best.
success_outcome | The default definition of success for this project type.
stage_zero_questions | Multiple-choice and short-answer questions asked before Observe begins.
primary_domains | Domains that should appear prominently in navigation and dashboards.
supporting_domains | Domains that remain available but are not emphasized by default.
default_overlays | Map overlays loaded frequently for this project type.
observe_objectives | Starter objectives for evidence gathering.
plan_objectives | Starter objectives for decisions, design, sequencing, and approval.
act_objectives | Starter objectives for assignment, execution, verification, maintenance, and feedback.
red_flags | Constraint checks that may require investigation, redesign, or rejection.
dashboard_widgets | Recommended command-center cards, progress trackers, and summary panels.
handoff_requirements | Minimum data required before progressing from one stage to the next.
[/TABLE]

Infrastructure level | Minimal; temporary; basic utilities; production infrastructure; guest/resident infrastructure; fully developed campus.

8. Detailed Project Type Templates
1. Homestead / Family Land Template
For a household-scale land project focused on food security, simple infrastructure, daily living, resilience, and manageable stewardship rather than maximum commercial production.

[TABLE]
Field | Template Specification
Template ID | PT-HOMESTEAD
Best fit | One household or family stewarding land for personal food, animals, shelter, water, energy, and long-term resilience.
Default success outcome | A realistic household land plan with safe access, water, food systems, basic infrastructure, and maintenance rhythms sized to the household capacity.
Primary domains | Vision & Project Intent; Land Base & Legal Context; Hydrology & Water; Soil & Subsurface; Plants, Crops & Food Systems; Animals, Livestock & Wildlife; Built Infrastructure; Energy, Materials & Resource Flows; Economics & Capacity; Monitoring, Records & Feedback.
Supporting domains | Climate & Microclimate; Topography & Landform; Ecology & Biodiversity; Access, Circulation & Logistics; People, Roles & Governance; Risk, Compliance & Suitability.
Default overlays | Zones; Water Flow; Soil Conditions; Access / Movement; Infrastructure / Utilities; Suitability; Timeline / Phasing; Stewardship Intensity; Risk / Compliance.
Stage Zero questions to ask | Primary household goals; desired food self-reliance percentage; animals desired; residence status; off-grid intensity; available weekly labour; budget range; tolerance for temporary infrastructure; public/guest access level.
Observe starter objectives | Verify dwelling/legal constraints; map water access and runoff; document existing food-growing areas; assess soil near daily-use zones; map current access, storage, and utility points; document animal/wildlife pressure.
Plan starter objectives | Choose Zone 0/1/2 layout; select first-year garden and animal areas; decide water storage/irrigation approach; phase infrastructure by budget and labour; set maintenance schedule.
Act starter objectives | Assign setup tasks for access, water, garden, animal shelter, compost, storage, and safety; verify completion with photos, receipts, and recurring checklists.
Red-flag / disqualifier checks | No legal dwelling path; unreliable water; severe access limitations; animal bylaws blocking desired livestock; excessive maintenance burden for available labour; flood/fire/contamination issues near living areas.
Recommended dashboard widgets | Homestead readiness score; household stewardship load; water/food/infrastructure dashboard; weekly chore rhythm; first-year buildout checklist.
Developer notes | Default to small, practical bundles. The system should prevent overbuilding and compare every proposed feature against household labour capacity.
[/TABLE]

9. Template Combination Logic
Many real projects are hybrids. OLOS should allow one primary template and optional secondary templates. The primary template defines the main dashboard and default workflow. Secondary templates add objective packs, overlays, and red-flag checks without replacing the main project logic.

[TABLE]
Hybrid Scenario | Recommended Template Setup
Family homestead with egg sales | Primary: Homestead. Secondary: Silvopasture / Livestock. Optional: Regenerative Farm if revenue becomes serious.
Ecovillage with food forest and retreats | Primary: Intentional Community. Secondary: Food Forest and Agritourism / Retreat.
Regenerative farm with workshops | Primary: Regenerative Farm. Secondary: Education / Demonstration Land.
Conservation land with occasional tours | Primary: Conservation / Restoration. Secondary: Education / Demonstration or Agritourism if visitor access is paid/structured.
Remote off-grid land with animals | Primary: Off-Grid Resilience. Secondary: Homestead and Silvopasture / Livestock.
Wellness retreat with gardens and lodging | Primary: Wellness Sanctuary or Agritourism depending on revenue focus. Secondary: Homestead, Food Forest, or Market Garden as needed.
[/TABLE]

10. Template Routing Rules

[TABLE]
Rule | Developer Behavior
Primary purpose wins | Choose the template based on the main outcome, not the prettiest feature. A farm with guest tours is still a farm if production is the main purpose.
Revenue increases compliance emphasis | If paid guests, food sales, lodging, or public events are selected, increase Risk / Compliance, Economics & Capacity, and People / Governance visibility.
Residency increases legal and infrastructure checks | If people will live on site, prioritize Land Base & Legal Context, Built Infrastructure, Water, Waste, Access, and Risk / Compliance.
Animals increase daily stewardship intensity | If livestock are selected, preload daily/weekly animal care, water, fencing, shelter, predator, welfare, and rotation checks.
Public access changes the map | If visitors are allowed, separate public, steward-only, restricted, emergency, and service routes.
Remote land changes the critical path | If off-grid or remote, access, water, heat, energy, waste, communication, and emergency readiness must appear before optional amenities.
Ecological sensitivity restricts action | If sensitive habitat, wetland, protected features, or restoration-only zones exist, Act should require approval before physical intervention.
[/TABLE]

12. Acceptance Criteria for Developer Implementation
A user can select one primary project type and optional secondary project types during Stage Zero.
Selecting a template changes domain priority, default overlays, starter objectives, red-flag checks, and dashboard widgets.
Templates never remove universal domains; they only change emphasis and defaults.
The map view loads overlays based on selected domain + objective, then modified by project type context.
Each template creates at least one starter objective in Observe, Plan, and Act.
Each starter objective includes checklist items, default overlays, evidence/proof fields, and completion criteria.
Hybrid templates do not duplicate objectives; they merge objective packs and warn when capacity exceeds available labour/budget.
Templates can be edited, disabled, or reweighted by the user or administrator.
Every template includes risk/compliance checks appropriate to its use case.
Every template feeds a clean handoff into the next stage rather than trapping information inside a static setup form.

[TABLE]
Final Product Principle / Project type templates should make OLOS feel like it understands the user’s land-based goal from the beginning. The user should not have to assemble the entire operating system manually. The template should load the right starting path, then let the land evidence refine the path over time.
[/TABLE]
