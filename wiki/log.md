- [2026-05-30 Рђћ B4 follow-up: per-block tooltip fade + reverse-in-flight](log/2026-05-30-b4-tooltip-perblock-fade-and-reverse.md) Рђћ **Branch.** `feat/atlas-permaculture`. Closes the last two still-deferred bullets from the 2026-05-29 exit-fade ADR: per-block exit fade in multi-host stacks, and reverse-in-flight animation when `activeUnion` returns non-null mid-exit. Fade machinery refactored from CSS keyframes (`@keyframes tooltipFadeIn` / `tooltipFadeOut`) to **CSS transitions** on opacity + transform; `.tooltip` mounts at `opacity: 0` and `.tooltip[data-visible='true']` flips to `opacity: 1` (the enter transition fires from a `useLayoutEffect` that sets `visible=true` on mount); `.tooltip[data-exiting='true']` takes opacity back to 0 with `ease-in` timing. New `.hostBlock` rule mirrors the same shape for per-block fades. `prefers-reduced-motion: reduce` override nulls all four selectors' `transition`. **Reverse-in-flight is free** with transitions: removing `data-exiting` mid-fade lets the transition interpolate from the current visible value back to 1, with no snap and no JS Рђћ making the 2026-05-29 monotonic-key remount pattern unnecessary. `HostCanopyUnionTooltip.tsx` exports a new `HostBlockEntry` interface (hostId + per-entry `phase: 'entering' | 'exiting'`); `HostBlock` renders `data-testid='host-block-${hostId}'`, `data-exiting='true'` when phase==='exiting', and an `onTransitionEnd` filtered by `propertyName==='opacity'` + `target===currentTarget` that fires a new `onEntryExited?: (hostId: string) => void` prop. Container's own `onTransitionEnd` filters by both `propertyName==='opacity'` and `target===currentTarget` so bubbled per-block transitionends don't fire `onExited`. **Merge-by-hostId.** `PlanDataLayers.displayedUnion` state shape replaces 2026-05-29's `key + entries: HostBlock[]` with per-entry phase tracking (`entries: HostBlockEntry[]`); the monotonic key is gone. The mirror `useEffect` rewrites the merge: kept hosts (in both old and new sets) carry forward with phase `entering` and refreshed data; dropped hosts (in old but not new) flip to phase `exiting` and stay in the array; new hosts are appended in MapLibre topmost-first order. The React `key` on the rendered map iterator is now `entry.hostId` (stable per host) so a block's in-flight transition isn't cancelled when siblings drop out and the array re-indexes. `onEntryExited` callback drops the hostId from the entries array Рђћ or clears the mirror entirely if the last block just finished fading. Safety timeout bumped 180 Рєњ 200 ms (120 ms transition + 80 ms slack to absorb event-loop jitter and the `prefers-reduced-motion` instant-change case where transitionend doesn't fire). **Test scaffolding.** New `fireTransitionEnd(el, propertyName)` helper Рђћ `@testing-library/dom`'s `createEvent.transitionEnd` passes init to the constructor without a fallback, and happy-dom's `TransitionEvent` constructor silently drops the `propertyName` init key, so the helper `Object.defineProperty`'s the value onto the event post-construction. Existing tests migrated from `fireEvent.animationEnd({ animationName: 'tooltipFadeOut' })` to the new helper. Two new tests: reverse-in-flight asserts that flipping `exiting` from true Рєњ false during re-render clears `data-exiting` and a subsequent opacity transitionend does NOT fire `onExited`; per-block exit asserts that an entry with `phase='exiting'` carries `data-exiting='true'`, that opacity transitionend fires `onEntryExited(hostId)`, and that sibling `entering` blocks + transform-property transitionends do not. **Why transitions, not WAAPI.** The 2026-05-29 ADR speculated about `element.animate()` + `animation.reverse()`. Transitions get the same behaviour with zero JS: they always interpolate from the current computed value, which is exactly what "reverse" means for a single-property fade. **Why merge by hostId.** When the active set shrinks, the dropped host has to stay in the displayed array long enough to fade; indices shift under set changes, so a key-by-index would cancel its transition when the array re-reconciled. Invariants preserved: `pointer-events: none`, single-pin model, ESC + tap-outside-to-dismiss, multi-feature fan-out (2026-05-27), edge-clamp + entries-aware `ESTIMATED_H` (counts every block including phase-`exiting` ones so a tooltip about to lose a block stays clamped conservatively Рђћ no flicker-reposition mid-fade), reduced-motion behaviour (was: no keyframe animation; now: no transition). 71/71 vitest green (8 tooltip + 63 agroforestry; +2 tooltip tests over the 2026-05-29 ship).
