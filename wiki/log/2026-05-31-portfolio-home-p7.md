Phase 7 of the OLOS Portfolio Home epic (`OLOS_Portfolio_Home_Spec_v1.0`, plan `olos-port-twinkly-peach`). P1–P6 (four-zone map, rails, stage tokens, climate util, cross-project relationships, cross-project Observe) were committed in code in prior sessions but not individually logged here; this is the first wiki record of the epic's later phases. Three steward decisions drove the phase ([[decisions/2026-05-31-atlas-portfolio-home-p7]]): **capability-based** role gating (never role-name literals), **full §6** navigation (project-count-aware landing + AppShell repoint), **full §3.3** card revamp.

**7.0 — Latent P5 role-key fix.** `PortfolioAtAGlanceRail` owner gate read `roleMap.get(project.id)` (local id) — always `undefined` for synced projects. Changed to `project.serverId`-keyed lookup, gated on `scopedRole == null /* local ⇒ owner */ || hasCapability(scopedRole, 'manage_members')`.

**7.1 — `portfolioAccess(project, roleMap)` helper (`portfolioModel.ts`).** Single source of truth for every §8 gate: returns `{ role, isOwnerTier, canEdit, isContractor }` where `role = serverId ? roleMap.get(serverId) ?? null : null`; `isOwnerTier = role == null || hasCapability(role, 'manage_members')` (local-only ⇒ owner); `canEdit = role == null || hasCapability(role, 'edit')`. Imports `hasCapability` from `@ogden/shared`. Also added `projectTypeBadges(p)` (shared by card + rail).

**7.2 — `PortfolioSummaryBar` (new `.tsx` + `.module.css`, mounted in `PortfolioDashboardView` above `.grid`).** BentoBox(flat) reading `usePortfolioStages` + `useProjectUrgency`: total projects, total area (Σ numeric `p.acreage`, dominant `units` → "ac"/"ha", "—" when none), per-stage count chips coloured via `STAGE_PAINT` (tappable → `onStageFilter`), open-divergences metric (Σ `breakdown.divergencesCritical + divergencesHigh`, tappable → `onToggleDiverged`, disabled at 0). `PortfolioDashboardView` owns `stageFilter` + `divergedOnly` state (`ordered` → `visible`) so the bar drives the grid (no new store); "No projects match filters" + Clear-filters when the filtered set is empty.

**7.3 — Full §3.3 `ProjectUrgencyCard` revamp.** Rebuilt as a BentoBox(elevated, padding none) composition with full-bleed stage colour bar (`STAGE_PAINT[stage].color` via inline `--stage-color`), serif name (`--font-serif`), primary/secondary type badges, stage pill + active-stratum label + Plan-progress bar, last-activity line, urgency-chip alert row (the old chip logic, preserved — not deleted), area, and an explicit **Open** CTA (whole-card click + resume-wizard-on-`draftWizard` retained). New required `stage` prop; `planProgress` + `role` passed from the parent. `RoleBadge` kept for non-steward roles. New `usePortfolioPlanProgress(projects)` hook computes stratum/objective progress for the whole grid in the parent (no per-card store subscriptions).

**7.4 — Access control §8 (capability-based).** New-project shown to all authed users (no `admin` role; creating ⇒ owner; removed the P1 "owner-only gating lands in P7" TODO without restricting). Relationship-create: owner-tier on that project only. Cross-project Observe compare: gated on `projects.some(p => portfolioAccess(p, roleMap).isOwnerTier)` — links hidden in `PortfolioProjectList` (new `canCompare` prop) and `DomainDetailHeader`, and `PortfolioObserveComparePage` short-circuits (after all hooks) to a read-only notice otherwise. Contractor guard: `usePortfolioContractorRedirect` (new) runs in the route **component** (roles are async), guarded by a `redirected` ref + `roleMap.size === 0` early-return; a contractor-somewhere / owner-tier-nowhere viewer is `navigate`d to their assigned project's Act surface (not per-project Home — avoids the PerProjectHomePage contractor empty-state dead-end).

**7.5 — Navigation model §6.** `landingRoute` (`routes/index.tsx`) `beforeLoad` rewritten to a **sync** project-count branch on `useProjectStore.getState().projects.filter(p => p.status !== 'archived')`: 0 → `/home`; 1 → `/v3/project/$id/home`; 2+ → `/v3/portfolio` (keeps `isAuthenticated()` guard; no role logic — async). `AppShell.tsx` "All Projects → `/v3/project`" link repointed to `/v3/portfolio` and relabelled "Portfolio" (kept the `!isHome && pathname !== '/new'` guard + aria-label).
