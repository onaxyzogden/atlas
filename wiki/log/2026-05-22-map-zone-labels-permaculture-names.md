# 2026-05-22 — Map zone labels show permaculture names, not "(seeded)"

**Branch.** `feat/atlas-permaculture`.

**Problem.** On the Plan map, zones rendered as "Z5 (seeded) — 837 ac", "Z2 (seeded) — 7.1 ac", etc. instead of the descriptive permaculture names. **Root cause:** the symbol label was the zone's raw stored `name`, and the ring-seed generator hardcoded that name as `` `Z${zLevel} (seeded)` `` (`ringSeedGenerator.ts`); the symbol layer in `PlanDataLayers.tsx` rendered `z.name` verbatim. The descriptive labels already existed (the Z-level picker in `ZoneLevelLayer.tsx`) but nothing mapped a zone's `permacultureZone` level to that label for the map. "(seeded)"/provisional status is shown **separately** by the dashed seed-line layer (driven by `seedProvenance`), so the name never needed to carry "(seeded)".

**Decisions (steward-chosen).** Keep the `Z#` prefix; **preserve any custom/user-renamed name**; fix **both** the display rendering (so zones already in localStorage relabel on reload — no migration) **and** the generator (so newly seeded zones store the clean name).

**Approach: single source of truth.** New `apps/web/src/lib/zones/permacultureLabels.ts` exports `PERMACULTURE_ZONE_LABEL` (Z0–Z5, the exact `ZONE_LEVELS` strings) and a pure `zoneDisplayLabel(z)` helper — returns the canonical label only when the name is auto/seeded (`/\(seeded\)\s*$/i`) or empty, otherwise the stored name (preserves "Home centre" and custom names). Wired into three call sites: `PlanDataLayers.tsx` label assignment (`label: zoneDisplayLabel(z)`), `ringSeedGenerator.ts` (seeded Z1–Z5 now store `PERMACULTURE_ZONE_LABEL[zLevel]`; Z0 stays "Home centre"), and `ZoneLevelLayer.tsx` (`ZONE_LEVELS` now derived from the canonical map so the picker and map can't drift apart again). Generator idempotency is unaffected — it keys off `seedProvenance` + `permacultureZone`, not the name.

**Verification.** 10/10 unit tests pass — 6 in `ringSeedGenerator.test.ts` (asserting Z1–Z5 store canonical names, Z0 = "Home centre") + 4 new in `permacultureLabels.test.ts` (relabel seeded/empty, preserve custom + "Home centre", fall through when no level). `tsc --noEmit` clean for touched files (3 pre-existing unrelated errors remain: `StepBoundary.tsx`, two `HostUnion*` tests). Plan map mounts with no runtime errors from the changed path. **Could not screenshot the steward's own seeded zones** — the preview browser profile has no stored zones (different profile than the session that produced the screenshot), so visual confirmation is deferred to the steward's reload.

**Observed, out of scope.** The Plan map logs a pre-existing MapLibre error from `DesignElementLayers.tsx` (`design-el-line.paint.line-width`: "Only one zoom-based step/interpolate subexpression may be used") — unrelated to this fix; flagged for separate attention.
