## Fix 1 — `waterRouter` TS7053 baseline error (`fabf3e38`, 1 file)

`apps/web/src/v3/observe/components/layers/ObserveAnnotationLayers.tsx` carried a typecheck baseline error: `subToggles[spec.toggleKey]` (lines ~1541/1565) raised **TS7053 (implicit any from index access)** because the `LayerSpec.toggleKey` `Exclude` union and the `subToggles` `Record` key set had **drifted out of sync**. The store added `waterRouter` (a Plan-stage downslope-flow/catchment overlay, `matrixTogglesStore` v14, default off) to `MatrixToggleKey`; the `subToggles` Record already excluded it (`Exclude<MatrixToggleKey, 'observeAnnotations' | 'sunPath' | 'zoneRings' | 'seededZones' | 'waterRouter'>`) but `LayerSpec.toggleKey` did **not**, so the two unions no longer matched and the index lookup widened to `any`.

**Fix:** add `'waterRouter'` to the `LayerSpec.toggleKey` Exclude union (lines ~156–162) so both unions resolve to the identical key set — `topography, sectors, zones, water, builtEnvironment, scheduledMoves`. **Type-only; zero runtime change.** Observe annotation specs never gate on Plan-stage-only keys (`sunPath`, `zoneRings`, `waterRouter`), so excluding it is correct. **Verified:** `npm run typecheck` (8GB heap) — the two `waterRouter` errors disappeared, baseline dropped 6→4.

## Fix 2 — hidden "Open Observation Needs" tray (`f2a88288`, 1 file)

Steward: the **"Open Observation Needs"** carousel (the bottom tray of the Observe Command Centre, route `observe/command-centre`) "is so low on page that it's hidden" — pushed below the viewport fold. **Root cause:** a **grid track / child-count mismatch** from parallel-session commit `996d2948` ("drop command-centre topbar, move Compass into module tabs"), which removed the separate `.topbar` element (so `.shell` now has **3** children — `<nav class=tabs>` / `<div class=body>` / `<div class=bottomTray>`) but left `grid-template-rows` at **4** tracks (`auto auto minmax(0,1fr) auto`). CSS grid auto-placement therefore put `.body` in the second `auto` track, where it sized to its **content** (mapRegion `min-height: 420px` + sidebar/rail) instead of flex-filling, and `.bottomTray` landed in the `1fr` track with track 4 left empty — so body + tabs consumed the viewport and the tray fell off the bottom edge.

**Fix (`apps/web/src/v3/command/ObserveCommandCentrePage.module.css`, lines ~629–635):** drop the now-removed topbar's track — `grid-template-rows: auto minmax(0, 1fr) auto` (tabs / body / tray), and tidy the block comment to drop "topbar". Now `.body` lands in `minmax(0,1fr)` (flex-fills, scrolls internally — it already had `min-height: 0; overflow: hidden`) and `.bottomTray` sits at the bottom in `auto`, fully visible. **CSS-only — no JSX/component/store/behaviour change**; the responsive `@media (max-width:1100px)` block only retunes columns, so it's unaffected. **Verified** via DOM bounding-box reads on live preview (:5200): tray bottom 463 ≤ window height 475, the "Open Observation Needs" label visible; `preview_screenshot` times out on the MapLibre WebGL canvas (**stated, not claimed**).

**Observed, flagged out of scope:** in the fixed ~475px preview viewport `.body` collapses toward 0px because the `auto` tray's natural height (~318px) + tabs + chrome (~427px) approaches the available shell height (threshold ~488px); all realistic screens clear it. Offered the steward an optional tray `max-height` or `.body` `min-height` clamp — awaiting their call.
