**Branch.** `feat/atlas-permaculture` (explicit-path slice commits; not pushed -- out-of-band rebase [[project-branch-rebase]], awaiting steward "go").

**Objective.** Merge the steward-supplied "OLOS Plan Stage Navigation UX Specification v1.1" + concept `.tsx` FUNCTIONALITY into the production Stratum Spine page while PRESERVING the current rounded/organic aesthetic. Steward's verbatim intent: *"I prefer the roundedness of what we have now however I appreciate the functionality of this proposed concept."* The concept's flat dark inline styling was discarded wholesale; every new element was built with the existing CSS Modules + `tokens.css` recipe (radius-md cards, radius-full pills, color-mix tinted surfaces, 120ms transitions, **Inter sans**, gold/sage/teal/amber palette). The real gap was UI surfacing plus one genuinely-absent flow: mid-project secondary-type addition (spec section 9), where `versionHistory` was never written. So the build split into Part A (surface existing data) + Part B (build the missing flow).
