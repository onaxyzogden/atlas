## Phase tags
P2, P3, MT (see `packages/shared/src/featureManifest.ts` for the authoritative item list)

## Purpose
Own the place/rotate/resize lifecycle for buildings and shelter-scale
structures on the project map: cabins, earthships, yurts, pavilions,
greenhouses, barns, workshops, storage/animal/compost/water buildings,
tent and glamping layouts, and MT-gated spiritual and communal
structures (prayer space, bathhouse, classroom). The section wraps a
footprint library with placement helpers, then layers P3 analytics on
top (setback/slope/solar/wind/view/privacy/noise warnings, walkability
and zone-relationship checks, build-order dependency logic,
infrastructure requirement + cost placeholder summary, alternate
footprint options).

## Redirect notice
The scaffolded folder is a placeholder for routing and manifest
registration. The live structure-placement implementation lives in
`apps/web/src/features/structures/` (footprint library + properties
modal) and remains the canonical owner of the P2 placement items.
MT prayer-space surfaces live in `apps/web/src/features/spiritual/`
(`PrayerSpaceAlignment.tsx`, `QiblaDisplay.tsx`). This section's
folder will become the composition layer that unifies those two
surfaces plus the P3 analytics added later — do not duplicate the
footprint library here.

## Known gotchas
- **MT gating.** `prayer-space, bathhouse, classroom placement` is
  tagged `MT` per the spec. The route handler uses
  `fastify.requirePhase('P2')` (the scaffolder's default) — when
  wiring real handlers, split this item onto a dedicated MT-gated
  route and confirm the web surface reads the per-item phase from
  the manifest, not the section's default phase. MT routes 404
  unless `ATLAS_MOONTRANCE=1`.
- **Prior-art split.** The P2 placement items are already partially
  implemented under `features/structures/`. Do not re-create the
  footprint library here — compose the existing module. Likewise the
  MT prayer-space surface already exists under `features/spiritual/`.
- **Rotation units.** `features/structures/footprints.ts` stores
  footprints centred at `[0,0]` in metres; placement translates via
  Turf.js. Any rotation helper added here must agree on radians vs
  degrees with the existing module.
- **Build order logic (P3)** is a graph problem, not a list — plan
  for a topological sort over structure → utility → access
  dependencies before implementing a UI.
- **Cost placeholder** is an explicit placeholder, not a real
  cost-model hook. Do not wire to Section 22 (Economic) without a
  design-review gate.
