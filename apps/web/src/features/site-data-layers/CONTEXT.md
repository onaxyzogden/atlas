## Purpose
Aggregates the external environmental datasets (elevation, hydrology, soils,
wetlands, land cover, climate, zoning) that describe a project site and
surfaces a Data Completeness Score on the project dashboard. The section
unifies Tier 1 public-data pipelines (USGS 3DEP, NHD, SSURGO, NWI, FEMA,
NLCD, NOAA) with Tier 2 user-supplied inputs (drone imagery, manual soil /
water tests, field notes) into a single layer catalog per project.

## Known gotchas
- Tier 1 adapters live under `apps/api/src/services/pipeline/adapters/` and
  are orchestrated by `DataPipelineOrchestrator.ts`; do not call adapters
  directly from routes — go through the orchestrator so `project_layers`
  rows are cached and versioned.
- SSURGO is US-only; `SoilGridsAdapter` covers the global fallback. The
  manifest item `soil-survey-ssurgo` is `partial` because international
  coverage still relies on SoilGrids.
- NHD / NLCD / NWI / NOAA coverage is US-centric; Ontario equivalents
  (OHN, AAFC Land Cover, ECCC Climate, Conservation Authorities) exist as
  separate adapters. Expect duplicate surfacing when project country = CA.
- Zoning overlays are jurisdiction-scoped: Ontario municipal registry is
  wired via `OntarioMunicipalAdapter`; other jurisdictions are planned.
- Phase-gating: P2 items (drone imagery, manual tests, habitat notes, etc.)
  should render as locked placeholders when `ATLAS_PHASE_MAX=P1`.
- `project_layers` stores both raster COG references and vector GeoJSON
  blobs; do not assume a single geometry shape in list views.
