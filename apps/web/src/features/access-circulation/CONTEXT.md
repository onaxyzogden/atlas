## Phase tags
P2, P3, MT (see `packages/shared/src/featureManifest.ts` for the authoritative item list)

## Purpose
Plan and analyze every way people, animals, vehicles, and services move across
a property. Owners draw and classify routes (main/secondary/emergency/service
access, pedestrian paths, trails, farm lanes, animal corridors, grazing routes,
accessible routes, quiet routes, arrival sequences). Over those routes the
section layers circulation intelligence: route slope analysis, conflict
detection between routes and zones, public vs private circulation layers,
parking/turning-radius/delivery checks, event flow, night-lighting safety,
and wayfinding. Feeds Section 9 (Structures) siting, Section 11 (Livestock)
grazing plans, and MT's guest-experience sequencing.

## Known gotchas
- **MT gating**: `arrival-sequence-design` and `quiet-circulation-routes`
  are tagged `MT`, not `P2`. They must be hidden unless
  `ATLAS_MOONTRANCE=1` — do not leak them into non-MT builds even though
  the underlying drawing UI already exists in `features/access/`.
- The canonical implementation folder is `features/access/`, not this
  folder. Do not duplicate components here — extend there and re-export.
  When the folder is renamed in a later pass, update `AccessCirculationPage.tsx`
  to mount the real panel instead of the "coming soon" stub.
- `PathType` in `pathStore.ts` is the source of truth for route
  classification (`main_road`, `emergency_access`, `service_road`,
  `pedestrian_path`, `animal_corridor`, `grazing_route`, `arrival_sequence`,
  `quiet_route`). New feature keys in this manifest entry must map onto
  those types — do not invent parallel enums.
- Route slope analysis depends on `useSiteData.terrain_analysis` being
  populated. If Site Assessment (Section 4) has not run, `SlopeWarnings`
  renders an empty state — surface the dependency clearly in the UI.
- Conflict detection in `accessAnalysis.ts` uses `@turf/turf` intersection
  primitives. Complex multi-polygon zones can be slow; memoize aggressively
  when expanding to public/private circulation layering.
- "Accessible route planning" (grade, width, surface) is **planned** — no
  existing surface. Do not conflate with the generic `pedestrian_path` type.
