## Phase tags
P1, P2 (see `packages/shared/src/featureManifest.ts` for the authoritative item list)

## Purpose
Provide the foundational map canvas for every other workflow in Atlas: choose
a basemap (satellite/aerial/topo/street/hybrid), toggle between 2D and
2.5D/3D terrain, render hillshade/contour/slope/aspect, overlay vector
layers (parcels, roads, waterbodies, buildings), run on-canvas
measurements (distance/area/elevation), and control layer
visibility/opacity/ordering. Every downstream section (hydrology,
structures, access, etc.) composes onto the chrome this section owns.

## Phase 3 additions (2026-04-23)
- `features/map/ViewModeSwitcher.tsx` ‚Äî 2D/2.5D/3D control. Wired into MapView
  top-left stack. 2.5D calls `map.easeTo({ pitch: 45 })`; 3D flips
  `is3DTerrain` ‚Üí mounts CesiumTerrainViewer.
- `features/map/CrossSectionTool.tsx` ‚Äî activates MapboxDraw line mode, POSTs
  the drawn LineString to `POST /api/v1/elevation/profile`, renders the
  returned samples as an inline SVG chart in a floating bottom panel.
- `features/map/ViewshedOverlay.tsx` (+ `ViewshedToggle`) ‚Äî fetches
  `GET /api/v1/basemap-terrain/:projectId/viewshed` and renders the stored
  viewshed GeoJSON as a translucent fill layer on MapCanvas.
- API additions: `POST /api/v1/elevation/profile` (bilinear line sampling
  against the country-appropriate DEM) and
  `GET /api/v1/basemap-terrain/:projectId/viewshed` (raw GeoJSON read-out).
- Shared: `ElevationProfileRequest` / `ElevationProfileResponse` zod schemas
  (`packages/shared/src/schemas/elevation.schema.ts`).
- Honest gap: the observer-point for the viewshed is still owned by the Tier-3
  pipeline ‚Äî there is no click-to-place UX yet, so users see the
  pre-computed viewshed only. Observer editing is a follow-up.

## Phase 4 additions (2026-04-23)
- `features/map/HistoricalImageryControl.tsx` ‚Äî Esri Wayback release picker.
  Fetches `s3-us-west-2.amazonaws.com/config.maptiles.arcgis.com/waybackconfig.json`
  once, parses release dates, and on selection adds a raster tile layer
  (`wayback-imagery`) on top of the basemap. Mirrors the parcel boundary as a
  dashed gold overlay so the parcel stays orientable.
- `features/map/SplitScreenCompare.tsx` (+ `SplitScreenToggle`) ‚Äî mounts a
  second maplibre instance in a resizable right pane. Camera sync is one-way
  (`primaryMap.on('move')` ‚Üí `right.jumpTo(...)`) with a reentry guard. The
  right pane is read-only: no draw, no edit. It mirrors the parcel boundary
  and a read-only snapshot of the primary map's MapboxDraw features (lines,
  polygons, points) via the `mirrorFeatures` prop fed by MapView.
- `store/mapStore.ts` adds `historicalRelease`, `splitScreenActive`,
  `splitScreenStyle` + setters.
- Honest gaps:
  - Wayback coverage is global but *not* every date has imagery for every
    parcel ‚Äî the tile server returns blanks in those cases. No empty-state
    UI yet.
  - Split-screen right pane has no measurement or draw tools ‚Äî it's a
    visual diff only. `mirrorFeatures` reflects what the user drew on the
    primary map, but they cannot interact with it.
  - Move sync is jump-based (not interpolated) ‚Äî during fast pans the
    compare pane can look choppy.

## Phase 5 additions (2026-04-23)
¬ß2 fully closed out ‚Äî remaining partials flipped to done.
- `features/map/OsmVectorOverlay.tsx` (+ `OsmVectorControls`) ‚Äî Overpass API
  fetches roads/waterways/buildings within the parcel bbox and paints them as
  toggleable GeoJSON layers. Each kind is independently toggled via
  `mapStore.osmLayersVisible`. Panel also exposes the global overlay-opacity
  slider.
- `features/map/MeasureTools.tsx` ‚Äî gained an **Elevation** mode: single-click
  anywhere on the map POSTs (lng, lat) to `POST /api/v1/elevation/point` and
  displays metres + feet. Distance and area modes still use Turf.js locally.
  Finally mounted in MapView ‚Äî was orphan before.
- `store/mapStore.ts` adds `overlayOpacity` (applied by historical raster,
  viewshed fill/line, OSM layers) and `osmLayersVisible` record.
- API: `POST /api/v1/elevation/point` ‚Äî small-bbox read + bilinear sample
  against the country-appropriate DEM, behind the same P2 gate as profile.
- Shared: `ElevationPointRequest` / `ElevationPointResponse` in
  `packages/shared/src/schemas/elevation.schema.ts`.
- Honest gaps:
  - Overpass rate-limits aggressively; the fetch is best-effort with no retry
    or local cache. Large parcels may hit the 25-second timeout.
  - Layer **ordering** UI is implicit ‚Äî OSM layers render above basemap and
    below draw/measurement in add-order; there's no drag-to-reorder panel.
    The spec said "ordering, opacity controls" ‚Äî ordering is de facto, opacity
    is explicit.
  - `EnvironmentOverlays` and `LayerPanel` remain orphaned. They targeted a
    Mapbox `composite` source that our MapTiler styles don't expose. OSM via
    Overpass replaces them functionally.
  - Elevation point tool reuses the `simple_select` draw mode but does not
    drop a persistent pin. Result is ephemeral text.

## Known gotchas
- Cesium and MapLibre run in the same page. Swap between them via the
  mode toggle ‚Äî do not mount both simultaneously (WebGL context pressure).
- Esri basemap tiles replaced Bing imagery (see commit d797dbf); attribution
  must stay wired.
- Viewshed and cross-section (P2) need elevation raster access, not just the
  sampling endpoint; plan for CoG reads before implementing.
- Historical imagery (P2) has no provider selected yet ‚Äî scope a data
  source before building UI.
- Split-screen / before-after (P2) implies two synchronized MapCanvas
  instances ‚Äî revisit the single-instance assumption above.
