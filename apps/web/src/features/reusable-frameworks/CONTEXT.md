## Purpose
Provide reusable, shareable **design frameworks** ‚Äî whole-project archetypes
(regen farm, retreat center, men's cohort campus, family-oriented community,
the OGDEN template, etc.) and saved fragments (feature bundles, rule sets,
hotspot sets, phase structures, cost models) ‚Äî so designers can start from a
vetted starting point instead of a blank canvas. "Huge for scaling." At P4
these frameworks graduate into a marketplace of curated/community-contributed
templates.

## Known gotchas
- **Slug collision / legacy code warning.** Batch 0 (Gap B) created real,
  non-stub code at `apps/web/src/features/templates/` and
  `apps/api/src/routes/templates/` for project-level templates
  (`project_templates` migration 013). That is a **different and narrower
  concern** than this section ‚Äî Batch 0 covers "snapshot a project, replay
  it as the starting state of a new project." This section covers
  reusable *design frameworks* (whole-archetype templates, saved rule sets,
  saved hotspot sets, etc.) and the eventual marketplace. The two are
  related but must not be conflated; the `reusable-frameworks` slug was
  chosen specifically to avoid colliding with the legacy `templates/` code.
  The manifest item `template-duplication-locking-governance` is the single
  overlap point ‚Äî it is tagged `status: 'partial'` because Batch 0's
  duplication flow already covers part of that surface.
- **MT gating.** Three items are MT-phase (`ogden-template`,
  `mens-cohort-campus-template`, `family-oriented-community-template`) and
  will 404 unless `ATLAS_MOONTRANCE=1`. The route handler currently gates
  at the section default `P3` ‚Äî when MT items get their own endpoints they
  must be registered behind `fastify.requirePhase('MT')` individually, not
  under the section default.
- **Marketplace (P4) is gated separately.** Marketplace UI and routes must
  live behind `fastify.requirePhase('P4')` and should not leak into any
  P3-max build.
