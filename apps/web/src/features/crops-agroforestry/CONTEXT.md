## Phase tags

## Purpose
Plan the productive planting layer of a property: orchards, row crops, garden
beds, food forests, and agroforestry bands (windbreaks, shelterbelts,
silvopasture, pollinator strips). Couples species suitability and
water-demand classification to the site's climate, soil, and frost layers
(Sections 3/6/7) and to the scored regen / water / ag layers (Section 4).
Owns the tree spacing calculator, canopy / root-overlap maturity
visualization, frost-safe orchard zoning, and access-to-harvest / irrigation
tie-in planning. Output feeds Section 15 (Timeline) for year-to-maturity
phasing and Section 22 (Economic) for yield placeholder estimates.

## Redirect notice
The scaffolded folder is a placeholder for routing and manifest registration.
Substantial legacy implementation already exists across four folders under
`apps/web/src/features/` and is the current canonical owner of the P2 items
flipped to `partial`:

- `features/crops/` — `CropPanel.tsx` drives orchard / row crop / garden bed /
  food forest polygon placement, wires into `useCropStore`, exposes the tree
  spacing calculator, species text input, and phase tagging.
- `features/planting/` — `plantingAnalysis.ts` + `plantSpeciesData.ts`
  provide species suitability filtering by site conditions, frost-safe
  planting window computation, companion-note surfacing, placement
  validation, and yield estimation.
- `features/forest/` — `forestAnalysis.ts` + `canopyLayerData.ts` cover
  silvopasture opportunity scoring, canopy layer data, existing vegetation
  breakdown, and tree-health proxies that feed the agroforestry /
  shelterbelt / silvopasture item.
- `features/nursery/` — `nurseryAnalysis.ts` + `propagationData.ts` own the
  nursery zone / propagation / readiness side of the "nursery, compost
  proximity planning" item; compost-proximity proper still lives in
  `features/structures/` + `features/utilities/`.

The live surfaces stay where they are. This section's folder becomes the
canonical route/manifest entry; downstream implementation should either
consolidate those four folders under `features/crops-agroforestry/` or
leave them in place and import from here. Decision deferred — see
"Known gotchas".

## Known gotchas
- Four-folder legacy split: consolidating `crops/` + `planting/` + `forest/` +
  `nursery/` under this section is a design decision deferred to
  implementation. Pick a convention before moving code — import paths,
  store keys, and CSS modules all diverge today.
- `features/crops/CropPanel.tsx` imports `CROP_TYPES` from
  `../livestock/speciesData.js`. Moving CropPanel without also restructuring
  that shared registry will break livestock placement.
- Tree spacing calculator currently lives inline inside `CropPanel.tsx`; it
  should become a pure helper under `packages/shared` once the Zod shape
  for spacing inputs is settled (Section 16 rules engine will also consume it).
- Yield estimates are placeholders by design — no real agronomic model
  until Section 22 lands. Do not advertise yield numbers as decisions.
- Canopy maturity / root-overlap and shade-succession (P3) need multi-year
  simulation state; plan for a project-scoped snapshot table before wiring
  the UI.
- Frost-safe zone detection depends on Section 6's frost-pocket raster —
  consume via the scored label, never by re-implementing the frost algorithm.
